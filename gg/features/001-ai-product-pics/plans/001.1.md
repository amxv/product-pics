## Database Schema

### Auth Tables (better-auth)

These tables are auto-generated by better-auth CLI. We extend the `user` table with the username plugin.

**user**
```typescript
{
  id: string (primary key)
  name: string
  email: string (unique)
  emailVerified: boolean
  image: string | null
  createdAt: timestamp
  updatedAt: timestamp
  username: string (unique) // Added by username plugin
}
```

**session**
```typescript
{
  id: string (primary key)
  userId: string (foreign key → user.id)
  expiresAt: timestamp
  token: string (unique)
  ipAddress: string | null
  userAgent: string | null
  createdAt: timestamp
  updatedAt: timestamp
}
```

**account**
```typescript
{
  id: string (primary key)
  userId: string (foreign key → user.id)
  accountId: string
  providerId: string
  accessToken: string | null
  refreshToken: string | null
  idToken: string | null
  expiresAt: timestamp | null
  password: string | null
  createdAt: timestamp
  updatedAt: timestamp
}
```

**verification**
```typescript
{
  id: string (primary key)
  identifier: string
  value: string
  expiresAt: timestamp
  createdAt: timestamp
  updatedAt: timestamp
}
```

### Application Tables

**batch**
```typescript
{
  id: string (primary key, nanoid)
  userId: string (foreign key → user.id, indexed)
  demographic: 'baby' | 'boy' | 'girl' (pgEnum)
  ageRange: string // "5-8" or "5"
  status: 'uploading' | 'uploaded' | 'processing' | 'completed' | 'failed' | 'partial' (pgEnum)
  totalImages: integer (default 0)
  completedImages: integer (default 0)
  failedImages: integer (default 0)
  createdAt: timestamp (default now())
  updatedAt: timestamp (default now())
  completedAt: timestamp | null
}
```

Indexes:
- `userId` (for batch history queries)
- `status` (for polling incomplete batches)
- `createdAt DESC` (for sorting)

**uploaded_image**
```typescript
{
  id: string (primary key, nanoid)
  batchId: string (foreign key → batch.id, indexed, onDelete: CASCADE)
  originalFilename: string
  r2Key: string (unique) // "uploads/{userId}/{batchId}/{id}.png"
  fileSize: integer // bytes
  mimeType: string
  status: 'pending' | 'uploaded' | 'failed' (pgEnum)
  errorMessage: text | null
  createdAt: timestamp (default now())
}
```

Indexes:
- `batchId` (for fetching all images in a batch)
- `r2Key` (unique, for R2 operations)

**generated_image**
```typescript
{
  id: string (primary key, nanoid)
  batchId: string (foreign key → batch.id, indexed, onDelete: CASCADE)
  uploadedImageId: string (foreign key → uploaded_image.id, indexed)
  r2Key: string | null // "generated/{userId}/{batchId}/{id}.png"
  status: 'pending' | 'processing' | 'completed' | 'failed' (pgEnum)
  retryCount: integer (default 0)
  background: string | null // "playground", "beach", etc.
  runpodJobId: string | null // Latest job ID
  errorMessage: text | null
  createdAt: timestamp (default now())
  completedAt: timestamp | null
}
```

Indexes:
- `batchId` (for batch progress tracking)
- `uploadedImageId` (one-to-one relationship)
- `status` (for polling processing images)

**runpod_job**
```typescript
{
  id: string (primary key, nanoid)
  generatedImageId: string (foreign key → generated_image.id, indexed)
  jobId: string (unique) // RunPod job ID
  status: 'in_queue' | 'in_progress' | 'completed' | 'failed' (pgEnum)
  prompt: text
  background: string
  submittedAt: timestamp (default now())
  completedAt: timestamp | null
  errorMessage: text | null
  resultUrl: string | null // RunPod CDN URL (temporary)
}
```

Indexes:
- `generatedImageId` (for tracking job history)
- `jobId` (unique, for status polling)
- `status` (for querying pending jobs)

### Validation Schemas (Zod)

```typescript
import { z } from 'zod';

// Age range validation
export const ageRangeSchema = z.string().refine(
  (val) => {
    // Single age: "5"
    if (/^\d+$/.test(val)) {
      const age = parseInt(val, 10);
      return age >= 0 && age <= 18;
    }
    // Age range: "5-8"
    if (/^\d+-\d+$/.test(val)) {
      const [min, max] = val.split('-').map(n => parseInt(n, 10));
      return min >= 0 && max <= 18 && min <= max;
    }
    return false;
  },
  { message: 'Age must be 0-18 or a range like "5-8"' }
);

// Batch creation validation
export const createBatchSchema = z.object({
  demographic: z.enum(DEMOGRAPHICS),
  ageRange: ageRangeSchema,
});

// Presigned URL request validation
export const presignedUrlRequestSchema = z.object({
  batchId: z.string().min(1),
  fileName: z.string().min(1),
  fileType: z.enum(SUPPORTED_FORMATS),
  fileSize: z.number().int().positive().max(20 * 1024 * 1024), // 20MB max
});
```

#### Patterns, Conventions, and Design Decisions to Reuse

**Authentication Pattern:**
- Use better-auth `username` plugin for username/password auth
- Server Components: `await auth.api.getSession({ headers: await headers() })`
- API Routes: `await auth.api.getSession({ headers: request.headers })`
- Client hooks: `useSession()` from auth-client

**Database Pattern:**
- Use Drizzle ORM with PostgreSQL
- Define pgEnums for all enum-like fields (demographic, statuses)
- Use `nanoid` for primary keys (shorter, URL-safe vs UUID)
- Cascade deletes for child records (uploaded_image, generated_image when batch is deleted)

**Type Safety Pattern:**
- Define enums as const arrays, infer types with `typeof`
- Use Drizzle's `InferSelectModel` and `InferInsertModel`
- Validate all API inputs with Zod schemas
- Export types from single source of truth (`lib/types.ts`)

**R2 Pattern:**
- Use AWS SDK v3 (`@aws-sdk/client-s3`)
- Configure with R2 endpoint (`https://{accountId}.r2.cloudflarestorage.com`)
- Set `forcePathStyle: true` for R2 compatibility
- Store file metadata in database, not just R2

#### Key Constraints to work within

- **better-auth requires 4 core tables**: user, session, account, verification
- **R2 requires CORS configuration** for presigned URL uploads
- **Next.js middleware runs on Edge**: Keep lightweight, avoid heavy DB queries
- **Environment variables**: All secrets must be in `.env` and never exposed to client
- **Drizzle migrations**: Must run `drizzle-kit generate` and `drizzle-kit migrate` for schema changes
